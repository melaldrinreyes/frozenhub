GOOGLE SIGN-IN / SIGN-UP FOR CUSTOMER ACCOUNTS
═══════════════════════════════════════════════════════════════════════════

This app uses Google Identity Services to create or sign in customer accounts.
No redirect callback is required.

What to configure
───────────────────────────────────────────────────────────────────────────
1. Create an OAuth client in Google Cloud Console.
2. Use application type: Web application.
3. Add these authorized JavaScript origins:
   - http://localhost:8080
   - https://frozenhub-pos.vercel.app
4. Copy the OAuth client ID.

Environment variables
───────────────────────────────────────────────────────────────────────────
Set the same client ID in both places:

- VITE_GOOGLE_CLIENT_ID
- GOOGLE_CLIENT_ID

Behavior
───────────────────────────────────────────────────────────────────────────
- Clicking "Continue with Google" creates a new customer account if the email is new.
- If the email already belongs to a customer account, it signs the user in.
- If the email belongs to admin/branch/rider/pos roles, the sign-in is rejected.
- Password is not required for Google-created customer accounts.

Notes
───────────────────────────────────────────────────────────────────────────
- The server verifies Google ID tokens before creating or signing in the account.
- The customer role is enforced server-side.
- You do not need a Google client secret for this flow.
